## 1. 架构概览

栾媛小工具是基于 Electron 的桌面应用，采用「主程序 + 插件」的架构。主程序与插件完全解耦，插件在运行时从插件目录动态加载。

```
┌─────────────────────────────────────────────────────────────┐
│                      Electron 主进程                         │
│  · 窗口管理 · 托盘 · 自定义协议 ly-plugin:// · IPC           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    渲染进程 (Vue 3 SPA)                       │
│  · 路由 (createMemoryHistory) · Element Plus · 插件加载器     │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              插件 (UMD，运行时从 ly-plugin:// 加载)            │
│  · App.js · style.css · 共享主程序 Vue/ElementPlus/Icon        │
└─────────────────────────────────────────────────────────────┘
```

| 层级 | 技术 |

## 4. 插件目录与路径

| 环境 | 插件目录 |
|------|----------|
| 开发 | `./plugins`（项目根目录） |
| 打包后 | `~/.ly/tools/plugins`（用户目录） |

主程序不依赖 `plugins` 目录中的任何代码。构建时若 `plugins` 不存在，会跳过插件构建；主程序可独立构建。

## 5. 运行时插件加载

### 5.1 自定义协议 ly-plugin://

主进程注册 `ly-plugin://` 协议，从插件目录提供静态文件：

- `ly-plugin://<pluginDir>/App.js`  → 插件主入口（UMD）
- `ly-plugin://<pluginDir>/style.css` → 插件样式

### 5.2 加载流程

1. 主进程 `getPluginList()` 扫描插件目录，读取各插件的 `manifest.json`
2. 路由根据插件列表动态注册 `path: manifest.route`
3. 访问插件路由时，`getPluginComponentLoader(pluginDir)` 被调用
4. 通过 IPC 获取 `ly-plugin://pluginDir/App.js` 的 URL
5. 注入 `style.css`（若存在），再通过 `<script>` 加载 `App.js`
6. 插件 UMD 使用全局 `Vue`、`ElementPlus`、`Icon`（由主程序在启动时暴露）

### 5.3 宿主环境暴露

主程序在 `main.js` 中通过 `exposePluginHost()` 设置：

```js
window.Vue = Vue
window.ElementPlus = ElementPlus
window.Icon = Icon
```

插件构建时使用这些作为 externals，不打包 Vue/ElementPlus/Icon。

## 6. 构建流程

### 6.1 主程序构建

```bash
npm run build
```

执行顺序：

1. `build-icon.js` - 生成图标
2. `build-plugins.js` - 构建插件（若 plugins 存在）
3. `vite build` - 构建主程序
4. `electron-builder` - 打包 Electron 应用
5. `apply-exe-icon.mjs` - 应用 exe 图标

### 6.2 插件构建

```bash
npm run build:plugins
```

将 `plugins/*/App.vue` 编译为 UMD 格式的 `App.js`，输出到各插件目录：

- `plugins/端口查杀/App.js`
- `plugins/端口查杀/style.css`
- ...

## 7. IPC 接口

| 接口 | 说明 |
|------|------|
| `get-plugin-list` | 获取已安装插件列表 |
| `get-plugin-entry-url` | 根据 pluginDir 获取插件入口 URL |
| `get-plugin-entry-url-by-id` | 根据 pluginId 获取插件入口 URL |
| `find-process-by-port` | 根据端口查询进程 |
| `kill-process` | 终止进程 |
| `save-file` | 保存文件对话框 |
| `get-platform` | 获取系统平台 |
| `install-plugin-from-market` | 从插件市场安装 |
| `open-lock-screen` / `close-lock-window` | 锁屏窗口 |
| `open-light-off-window` / `close-light-off-window` | 关灯窗口 |
| `open-lobster-window` / `close-lobster-window` | 养龙虾窗口 |
| ... | ... |

## 8. 路由

- 使用 `createMemoryHistory`，无 URL 哈希
- 首页：`/`
- 插件市场：`/plugin-market`
- 插件路由：由 `manifest.route` 动态注册（如 `/port-killer`）

## 9. 独立窗口

部分插件支持独立窗口（如养龙虾、关灯），通过 `?pluginId=xxx` 加载：

- `?pluginId=lobster` → 养龙虾
- `?pluginId=lock-screen-light-off` → 关灯

主进程创建窗口时传入对应 URL，渲染进程检测到 `pluginId` 后直接加载该插件并挂载。
