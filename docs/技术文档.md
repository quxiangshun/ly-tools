## 1. 架构概览

栾媛小工具是基于 Electron 的桌面应用，采用「主程序 + 插件」的架构。主程序与插件完全解耦，插件在运行时从插件目录动态加载。

```
┌─────────────────────────────────────────────────────────────┐
│                      Electron 主进程                         │
│  · 窗口管理 · 托盘 · 自定义协议 ly-plugin:// · IPC           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    渲染进程 (Vue 3 SPA)                       │
│  · 路由 (createMemoryHistory) · Element Plus · 插件加载器     │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              插件 (UMD，运行时从 ly-plugin:// 加载)            │
│  · App.js · style.css · 共享主程序 Vue/ElementPlus/Icon        │
└─────────────────────────────────────────────────────────────┘
```

| 层级 | 技术 |


## 4. 构建流程

### 4.1 主程序构建

```bash
npm run build
```

- **主程序构建不包含插件**。打包后的应用为纯净主程序，不含任何插件。
- 插件目录为 `~/.ly/tools/plugins`（用户目录），由用户通过插件市场安装。

执行顺序：

1. `build-icon.js` - 生成图标
2. `vite build` - 构建主程序
3. `electron-builder` - 打包 Electron 应用
4. `apply-exe-icon.mjs` - 应用 exe 图标

### 4.2 插件构建

```bash
npm run build:plugins              # 构建所有插件
npm run build:plugins -- 秀恩爱    # 仅构建指定插件
```

从 `plugins-ext/` 构建到 `plugins/`，并为每个插件生成 zip。

### 4.3 插件市场上架

如需在插件市场上架，需将 `plugins/*.zip` 上传至：

```
http://39.106.39.125:9999/tools/plugins/
```

**插件开发、目录路径、加载机制、manifest、故障排查等详见 [插件开发.md](./插件开发.md)**。

## 5. IPC 接口

| 接口 | 说明 |
|------|------|
| `get-plugin-list` | 获取已安装插件列表 |
| `get-plugin-entry-url` | 根据 pluginDir 获取插件入口 URL |
| `get-plugin-entry-url-by-id` | 根据 pluginId 获取插件入口 URL |
| `find-process-by-port` | 根据端口查询进程 |
| `kill-process` | 终止进程 |
| `save-file` | 保存文件对话框 |
| `get-platform` | 获取系统平台 |
| `install-plugin-from-market` | 从插件市场安装 |
| `open-lock-screen` / `close-lock-window` | 锁屏窗口 |
| `open-light-off-window` / `close-light-off-window` | 关灯窗口 |
| `open-shatter-window` / `close-shatter-window` | 碎屏屏保窗口 |
| `open-lobster-window` / `close-lobster-window` | 养龙虾窗口 |
| ... | ... |

## 6. 路由

- 使用 `createMemoryHistory`，无 URL 哈希
- 首页：`/`
- 插件市场：`/plugin-market`
- 插件路由：由 `manifest.route` 动态注册（如 `/port-killer`）
