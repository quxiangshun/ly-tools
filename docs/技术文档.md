## 1. 架构概览

栾媛小工具是基于 Electron 的桌面应用，采用「主程序 + 插件」的架构。主程序与插件完全解耦，插件在运行时从插件目录动态加载。

```
┌─────────────────────────────────────────────────────────────┐
│                      Electron 主进程                         │
│  · 窗口管理 · 托盘 · 自定义协议 ly-plugin:// · IPC           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    渲染进程 (Vue 3 SPA)                       │
│  · 路由 (createMemoryHistory) · Element Plus · 插件加载器     │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              插件 (UMD，运行时从 ly-plugin:// 加载)            │
│  · App.js · style.css · 共享主程序 Vue/ElementPlus/Icon        │
└─────────────────────────────────────────────────────────────┘
```

## 4. 插件目录与路径

| 环境 | 插件目录 |
|------|----------|
| 开发 | `./plugins`（由 build:plugins 从 plugins-ext 构建输出） |
| 打包后 | `~/.ly/tools/plugins`（用户目录） |

源码在 `plugins-ext/`，执行 `npm run build:plugins` 会构建到 `plugins/` 并生成各插件的 zip 包；主程序从 `plugins/` 加载。

## 5. 运行时插件加载

### 5.1 自定义协议 ly-plugin://

主进程注册 `ly-plugin://` 协议，从插件目录提供静态文件。完整定义见 [ly-plugin协议定义.md](./ly-plugin协议定义.md)。

**URL 格式**：`ly-plugin://plugin/<encodeURIComponent(pluginDir)>/<file>`

| 示例 | 说明 |
|------|------|
| `ly-plugin://plugin/%E7%AB%AF%E5%8F%A3%E6%9F%A5%E6%9D%80/App.js` | 插件主入口（UMD） |
| `ly-plugin://plugin/%E7%AB%AF%E5%8F%A3%E6%9F%A5%E6%9D%80/style.css` | 插件样式 |

> **说明**：pluginDir 必须放在 path 中，避免 host 编码解析问题导致加载失败。

**协议处理流程**（主进程 `protocol.handle`）：

```
请求 URL: ly-plugin://plugin/<encoded>/App.js
    │
    ▼
解析 pathname → parts = [encoded, "App.js"]
    │
    ▼
pluginDir = decodeURIComponent(parts[0])
file = parts.slice(1).join("/")
    │
    ▼
filePath = pluginsRoot/pluginDir/file
    │
    ▼
校验：filePath 在 pluginsRoot 内 且 文件存在
    │
    ▼
读取文件 → 按扩展名设置 MIME → 返回 Response
```

**MIME 映射**：`.js` → `application/javascript`，`.css` → `text/css`，`.json` → `application/json`

### 5.2 加载流程

1. 主进程 `getPluginList()` 扫描插件目录，读取各插件的 `manifest.json`
2. 路由根据插件列表动态注册 `path: manifest.route`
3. 访问插件路由时，`getPluginComponentLoader(pluginDir)` 被调用
4. 渲染进程 IPC 调用 `getPluginEntryUrl(pluginDir)` 获取 `ly-plugin://plugin/encoded/App.js`
5. 注入 `<link href="ly-plugin://plugin/encoded/style.css">`，再通过 `<script src="...">` 加载 App.js
6. 协议处理器拦截请求，从 `plugins/<pluginDir>/` 读取文件并返回
7. App.js 执行后设置 `window.__LY_PLUGIN__`，加载器读取并 resolve 为 Vue 组件
8. 插件 UMD 使用全局 `Vue`、`ElementPlus`、`Icon`（由主程序在启动时暴露）

### 5.3 宿主环境暴露

主程序在 `main.js` 中通过 `exposePluginHost()` 设置：

```js
window.Vue = Vue
window.ElementPlus = ElementPlus
window.Icon = Icon
```

插件构建时使用这些作为 externals，不打包 Vue/ElementPlus/Icon。

## 6. 构建流程

### 6.1 主程序构建

```bash
npm run build
```

执行顺序：

1. `build-icon.js` - 生成图标
2. `vite build` - 构建主程序
3. `electron-builder` - 打包 Electron 应用
4. `apply-exe-icon.mjs` - 应用 exe 图标

### 6.2 插件构建

```bash
npm run build:plugins              # 构建所有插件
npm run build:plugins -- 秀恩爱    # 仅构建指定插件
```

从 `plugins-ext/` 构建到 `plugins/`，并为每个插件生成 zip：

- `plugins/端口查杀/`：manifest.json、App.js、style.css 等
- `plugins/端口查杀.zip`：用于插件市场分发
- ...

## 8. 路由

- 使用 `createMemoryHistory`，无 URL 哈希
- 首页：`/`
- 插件市场：`/plugin-market`
- 插件路由：由 `manifest.route` 动态注册（如 `/port-killer`）

## 9. 独立窗口

部分插件支持独立窗口（如养龙虾、关灯），通过 `?pluginId=xxx` 加载：

- `?pluginId=lobster` → 养龙虾
- `?pluginId=lock-screen-light-off` → 关灯

主进程创建窗口时传入对应 URL，渲染进程检测到 `pluginId` 后直接加载该插件并挂载。

## 10. 故障排查

### 10.1 插件加载失败

**现象**：点击插件卡片后提示「插件加载失败」。

**原因**：`ly-plugin://` 协议 URL 格式问题。若将 pluginDir（含中文）放在 URL 的 host 部分（如 `ly-plugin://%E7%AB%AF.../App.js`），部分环境下 `protocol.handle` 解析 hostname 时会出现异常，导致 404。

**解决方案**：使用 path 格式，将 pluginDir 放在 path 中：

- 正确：`ly-plugin://plugin/<encodeURIComponent(pluginDir)>/App.js`
- 错误：`ly-plugin://<encodeURIComponent(pluginDir)>/App.js`

协议处理器从 `u.pathname` 解析：`/encodedDir/file` → `pluginDir`、`file`。
