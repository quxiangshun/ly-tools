## 1. 功能说明

本插件在 **Electron 桌面版**中打开本地 `.md` / `.markdown` 等文件，使用 **marked** 将 Markdown 渲染为 HTML，支持：

| 能力 | 说明 |
|------|------|
| 多标签 | 同时打开多个文件；标签页与当前选中标签可持久化到 `localStorage` |
| 最近打开 | 最多 **20** 条路径；支持单条从历史中移除 |
| 会话恢复 | 启动时用 `filterExistingFiles` 剔除已删除文件后再恢复标签 |
| 文档目录（TOC） | 从 `#`～`######` 标题生成侧栏目录；可收起侧栏；分栏模式下点击目录同时滚动编辑区与预览区 |
| 视图 | **预览** / **编辑** / **左右分栏** / **上下分栏**；与侧栏状态一并持久化 |
| 搜索 | 关键词、上一个/下一个、区分大小写；预览区高亮，当前命中与滚动；编辑区同步选中 |
| 持久化 | 仅内存编辑，**不自动写回磁盘** |

**Web 环境**（无 Electron）下无法打开本地文件，界面会提示「请在 Electron 桌面版中使用」。

## 2. 目录内文件

| 文件 | 说明 |
|------|------|
| `manifest.json` | 插件元数据：`id`、`route`、卡片展示、`version`、`minLyToolsVersion` 等 |
| `App.vue` | 插件界面与逻辑（单文件组件） |
| `package.json` | 插件专用依赖（当前为 **marked**），供 Vite 构建时解析并打入 UMD；**不**写在主程序根目录 `package.json` |
| `开发说明.md` | 本文档；构建后会复制到 `plugins/Markdown阅读/` |

构建产物（在项目根目录执行 `npm run build:plugins -- Markdown阅读` 后，输出到 **`plugins/Markdown阅读/`**）还包括：

- `App.js`、`style.css`（由 `App.vue` 编译生成）
- `node_modules/`（若 `package.json` 含 `dependencies`，构建脚本会在输出目录执行 `npm install`）

本插件**不**使用 `electronAPI.plugin.invokeMain` 或插件目录下的 Node 主进程脚本；文件读写与对话框均通过主程序已提供的 **`electronAPI.openFileDialog` / `readTextFile` / `filterExistingFiles`** 完成（见 [§4](#4-与主程序的约定)）。

## 3. 环境要求

| 项目 | 要求 |
|------|------|
| 栾媛小工具仓库 | 已克隆，能执行 `npm install` |
| 主程序版本 | **≥ `manifest.json` 中的 `minLyToolsVersion`**（当前为 **1.2.1**） |
| 运行环境 | 开发/调试 **Markdown 阅读**需 **Electron 桌面版**（`npm run dev` 启动的应用） |

`minLyToolsVersion`：**1.2.1** 对应主程序提供 **文件类 IPC**（`open-file-dialog`、`read-text-file`、`filter-existing-files` 等；预加载见 `electron/preload.js`）。若用户主程序版本过低，该插件**不会出现在首页**，不会加载。

## 4. 与主程序的约定

### 4.1 渲染进程 API（本插件直接使用）

| `electronAPI` 方法 | 用途 |
|--------------------|------|
| `openFileDialog({ title?, multiSelections?, filters? })` | 打开文件对话框（可多选） |
| `readTextFile(filePath)` | UTF-8 读文本；单文件最大 **20MB**（主进程限制） |
| `filterExistingFiles(paths)` | 按顺序返回仍存在的文件绝对路径 |

插件内通过 `window.electronAPI` 调用；若需核对签名与 IPC 通道，见 **[docs/插件开发.md](../../docs/插件开发.md) §5**。

### 4.2 可选 API（文档已暴露，按需使用）

| 方法 | 说明 |
|------|------|
| `listMarkdownInDir(dirPath)` | 列出目录内 Markdown 文件（非递归） |

### 4.3 与「数据库同步」类插件的差异

- **数据库同步**：业务在插件目录的 **`db-sync.js`**，经 **`electronAPI.plugin.invokeMain`** 调用。

- **Markdown 阅读**：不增加主进程脚本；仅使用主程序已提供的 **`openFileDialog` / `readTextFile` / `filterExistingFiles`** 等渲染进程 API。

## 5. `manifest.json` 要点

| 字段 | 本插件示例 | 说明 |
|------|------------|------|
| `id` | `markdown-reader` | 唯一标识，勿与仓库内其他插件冲突 |
| `route` | `/markdown-reader` | 路由路径 |
| `title` / `icon` / `description` / `color` | 见仓库内文件 | 首页卡片展示 |
| `version` | 与 `package.json` 的 `version` 建议一致 | 便于发版与排查 |
| `minLyToolsVersion` | `1.2.1` | 低于此版本主程序不加载本插件 |
| `supportedOS` | `win64`、`linux`、`darwin` | 按实际测试情况填写 |

修改 `manifest.json` 后需重新构建插件（见下文）。

## 6. 依赖与构建原理

1. **`marked`** 在 **`plugins-ext/Markdown阅读/package.json`** 的 `dependencies` 中声明。
2. 在项目根目录执行 **`npm run build:plugins -- Markdown阅读`** 时：
   - 在插件源码目录安装依赖（供 Vite 解析）；
   - 将 `App.vue` 编译为 **UMD** 的 `App.js`，样式为 `style.css`；
   - 复制 `manifest.json`、`开发说明.md` 等非排除文件到 **`plugins/Markdown阅读/`**；
   - 在输出目录执行 **`npm install --omit=dev`**，将运行时依赖装入 **`plugins/Markdown阅读/node_modules`**（随 zip 分发时可保留，便于与主程序约定一致）。

构建脚本路径：`scripts/build-plugins.js`。

## 7. 本地开发 — 完整步骤（从零到可调试）

按顺序执行：

1. **安装主程序依赖**（仓库根目录）  
   ```bash
   npm install
   ```

2. **构建本插件**（仓库根目录）  
   ```bash
   npm run build:plugins -- Markdown阅读
   ```  
   确认生成 **`plugins/Markdown阅读/App.js`**、`style.css`、`manifest.json` 等。

3. **启动主程序**  
   ```bash
   npm run dev
   ```

4. 在首页进入 **「Markdown 阅读」**，使用 **「打开 Markdown」** 选择本地文件，验证预览、标签、目录、搜索等。

5. **修改插件源码后**（`App.vue`、`manifest.json`、`package.json` 等），**必须**再次执行第 2 步，然后刷新或重启 Electron，改动才会生效。

## 8. 实现要点（便于二次开发）

| 项目 | 说明 |
|------|------|
| 会话存储 key | `localStorage` 键名：`ly-markdown-reader:v1`（见 `App.vue` 内 `STORAGE_KEY`） |

| 场景 | 做什么 |
|------|--------|
| **本地自测** | 执行 **§7** 的构建 + `npm run dev` |

## 11. 常见问题

### Q: 首页「Markdown 阅读」插件不显示？

- 主程序版本是否 **≥ `minLyToolsVersion`**（当前为 **1.2.1**）？
- 当前操作系统是否在 `manifest.json` 的 `supportedOS` 中？

### Q: 提示「请在 Electron 桌面版中使用」？

- 在浏览器中打开主程序时无 `electronAPI`，属预期；请用 **`npm run dev`** 或安装包启动桌面版。

### Q: 修改插件代码后不生效？

- 必须重新执行 **`npm run build:plugins -- Markdown阅读`**；`App.js` 由构建生成，不是直接编辑。

### Q: 与主程序 `electronAPI` 不一致？

- 以 **`electron/preload.js`** 为准；文档见 **[docs/插件开发.md](../../docs/插件开发.md) §5**。

### Q: 找不到本文件 `开发说明.md`？

- 源码位置：**`plugins-ext/Markdown阅读/开发说明.md`**（本仓库路径）。
- 构建后位置：**`plugins/Markdown阅读/开发说明.md`**（需先执行 `npm run build:plugins -- Markdown阅读`）。
