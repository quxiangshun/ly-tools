# 数据库同步插件 — 开发说明

Copyright (C) 2025 屈想顺. Licensed under AGPL-3.0.

## 1. 功能说明

本插件实现 **源数据库 A → 目标数据库 B** 的同步（同类型数据库之间）：将左侧配置的源库表结构与数据写入右侧目标库，**不是**单独的「导出文件」功能。

当前支持：**MySQL → MySQL**、**PostgreSQL → PostgreSQL**、**SQLite 文件 → SQLite 文件**。同步会覆盖目标库中与源库对应的内容，使用前务必确认连接信息与备份。

## 2. 目录内文件

| 文件 | 说明 |
|------|------|
| `manifest.json` | 插件元数据（`id`、`route`、卡片展示等）；**`minLyToolsVersion`** 声明所需主程序最低版本（见下节） |
| `App.vue` | 界面：源库 / 目标库连接信息与「将 A 同步到 B」操作 |
| `db-sync.js` | 主进程执行的同步逻辑，由 `electronAPI.plugin.invokeMain` 调用 |
| `package.json` | 本插件专用 npm 依赖（`mysql2`、`pg`、`sql.js`），**不**写在主程序根目录 `package.json` |
| `开发说明.md` | 本文档；构建后会复制到 `plugins/数据库同步/` |

构建产物（`npm run build:plugins` 后位于 `plugins/数据库同步/`）还包括：`App.js`、`style.css`，以及执行 `npm install` 生成的 `node_modules/`。

### 对栾媛小工具主程序（框架）版本的要求

本插件依赖主程序提供的 **`electronAPI.plugin.invokeMain`** 与 IPC **`invoke-plugin-main`**。在 **`manifest.json`** 中已声明：

```json
"minLyToolsVersion": "1.2.0"
```

含义：用户安装的栾媛小工具版本（**关于**对话框或安装目录下应用版本）须 **≥ 1.2.0**。若低于该版本，主程序加载插件列表时会**不加载**本插件，首页**不会出现**「数据库同步」卡片。  
分发 zip 时可在说明中写明：**需要栾媛小工具 ≥ 1.2.0**。若你以后只面向更新后的主程序，可把 `minLyToolsVersion` 提高到新版本号。

## 3. 与主程序的约定

- 渲染进程通过 **`window.electronAPI.plugin.invokeMain('db-sync', 'db-sync', 'syncDatabases', payload)`** 或 **`…, 'inspectConnection', payload)`** 调用主进程。
- 主进程使用通用 IPC **`invoke-plugin-main`**，按 `manifest.id`（`db-sync`）定位插件目录，用 **`createRequire(插件目录/package.json)`** 加载 `db-sync.js`，并执行对应方法（如 **`syncDatabases(payload, log)`**；第二参数 `log` 由主进程注入，用于推送执行日志）。主程序 **≥ 1.2.0** 时每次调用前会清除该脚本的 `require` 缓存，便于热更新 `db-sync.js`。
- 不在主程序中为本插件增加专用 IPC；通用约定见项目根目录 **`docs/插件开发.md`**（§5.2、§3.5）。

## 4. `db-sync.js` 接口

- **导出**：`module.exports = { syncDatabases, inspectConnection }`
- **`syncDatabases(payload, log)`**  
  - `payload.source` / `payload.target`：均含 `type`（`mysql` | `postgresql` | `sqlite`）及连接字段。  
  - `log`：可选；为函数时在关键步骤写入进度（PostgreSQL 对 `json`/`jsonb` 列会做规范化，避免非法字符串导致 `invalid input syntax for type json`）。  
  - 返回：`{ success: boolean, message?: string, ... }`（失败时 `success: false`）。  
  - 页面侧在 **`invokeMain` 前** 订阅 **`electronAPI.plugin.onMainLog`** 可在界面展示上述日志。
- **`inspectConnection(payload)`**（不触发同步）  
  - `payload`：含 `type` 及单端连接字段，可选 `side`（`source` | `target`）。  
  - 返回：`{ success: true, info: object }` 或 `{ success: false, message }`。

## 5. 依赖与构建

1. 在 **`plugins-ext/数据库同步/`** 维护 `package.json` 的 `dependencies`。
2. 在项目根目录执行：**`npm run build:plugins -- 数据库同步`**  
   - 会编译 `App.vue`，复制本目录文件，并在 **`plugins/数据库同步/`** 下执行 **`npm install --omit=dev`**（仅当存在非空 `dependencies` 时）。
3. 启动主程序：**`npm run dev`**（需先完成上一步，保证 `plugins/数据库同步/db-sync.js` 与 `node_modules` 存在）。

## 6. 本地使用（开发自测）

1. 完成 **§5** 的构建，确保 **`项目根目录/plugins/数据库同步/`** 下存在 `App.js`、`manifest.json`、`db-sync.js`、`package.json`、`node_modules/`。
2. 运行 **`npm run dev`** 启动栾媛小工具（Electron）。
3. 在首页进入 **「数据库同步」**，填写源库 A、目标库 B，使用 **「将 A 同步到 B」**。

开发时修改了插件源码后，需重新执行 **`npm run build:plugins -- 数据库同步`** 再刷新应用。

## 7. 发布上架

### 7.1 生成安装包

构建脚本会在 **`plugins/数据库同步.zip`** 生成可分发包（内容为 **`plugins/数据库同步/`** 整目录，含 **`node_modules`**）。  
**请勿**删掉 zip 里的 `node_modules`，否则用户机器上若未手动执行 `npm install`，主进程脚本将无法 `require` 数据库驱动。

### 7.2 上传到插件市场

项目 **README** 中说明：若需上架插件市场，将 **`plugins/*.zip`** 上传到：

`http://39.106.39.125:9999/tools/plugins/`

（具体上传方式以你方服务器/运维流程为准。）

### 7.3 市场列表 `index.json`（可选）

若市场使用 **`index.json`** 做中文名映射，可增加一条，例如：

```json
{ "filename": "数据库同步.zip", "title": "数据库同步" }
```

zip **文件名**决定用户安装后的**插件目录名**（去掉 `.zip`）；`manifest.json` 里的 **`id`**（如 `db-sync`）可与目录名不同，路由与 `invokeMain` 以 **`id`** 为准。

## 8. 终端用户安装与使用

### 8.1 安装方式（对照）

| 方式 | 说明 |
|------|------|
| **应用内插件市场** | 在栾媛小工具中打开 **插件市场**，选择本插件对应的 zip 安装；安装程序会将插件解压到用户目录 **`~/.ly/tools/plugins/<插件目录名>/`**。 |
| **手动安装** | 将 **`数据库同步.zip`** 解压到上述 **`plugins`** 目录。解压后须能访问到 **`.../plugins/数据库同步/manifest.json`**（或与主程序支持的「zip 顶层即单目录」结构一致）。 |

**Windows 上用户目录示例**：`C:\Users\<用户名>\.ly\tools\plugins\`  
**macOS / Linux**：`~/.ly/tools/plugins/`

### 8.2 使用步骤

1. 启动 **栾媛小工具**（安装版或开发版均可）。
2. 在首页进入 **「数据库同步」**。
3. 选择数据库类型，填写 **源库 A**、**目标库 B**，在确认对话框中同意后执行 **「将 A 同步到 B」**。

**路径说明**：已打包的主程序从 **`~/.ly/tools/plugins`** 加载插件；开发时从项目根目录的 **`plugins/`** 加载。逻辑一致，仅目录不同。
